# ***cloudflare***

域名托管在cloudfare之后，clareflare就会代管这些域名。所有通过域名进行访问的流量都会经过cloudflare，再由cloudflare进行转发至个人服务器。保护个人服务器不会收到攻击而瘫痪。

在cloudflare上还可以免费解析域名，免费提供cdn和ssh证书服务（小黄云），浏览器向网站请求服务时，网站会发给浏览器一个证书，握手成功后访问就会变成https加密，数据会通过加密通道传输，不然很容易被第三方抓包，被窃取数据。

# ***ngiex***

在解析完自己的域名后（一般使用“A”将子域名解析到自己服务器ip地址，然后打开终端ping一下），解析成功就可以用域名访问自己的IP地址了，但是默认访问的是80端口，这时候需要用neigx来将所需端口代理到指定的域名上了，于是就可以将子域名和docker部署网站一一对应起来。
